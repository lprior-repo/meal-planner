# Big structured SQL data: Postgres (Supabase, Neon.tech)

> **Context**: import DocCard from '@site/src/components/DocCard';

This page is part of our section on [Persistent storage & databases](./meta-11_persistent_storage-index.md) which covers where to effectively store and manage the data manipulated by Windmill. Check that page for more options on data storage.

For Postgres databases (best for structured data storage and retrieval, where you can define schema and relationships between entities), we recommend using Supabase or Neon.tech.
